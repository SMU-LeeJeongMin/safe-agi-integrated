# 합성 세션 데이터 생성 (정답 근거용)

## 1. 목적과 배경

### 왜 가상 데이터가 필요한가
safe_db 데이터 인벤토리 점검 결과, 의미 있는 생체 신호를 가진 유효 F1 세션은 단 1건(`97d67527...`, 116분)뿐이다. `inference_results`·`accident_history`·`environment_logs` 등은 비어 있다. 모델 학습·OPTUNA 피처 탐색에는 다수 세션이 필요하나 실측이 절대적으로 부족하므로, 실측 1세션을 뼈대로 정상/이상 다수 세션을 가상 생성한다.

### 정답 레이블 설계와의 연결
F1 정답 레이블 설계에서 정한 정답 기준이 "타당하게 도출되는" 세션을 만드는 것이 이 단계의 임무다. 즉 규격 판정 로직(과부하 5분·SpO₂ 등급·누적 90분)이 각 상황에서 의도한 위험 등급을 자연스럽게 뱉도록 신호를 설계한다.

---

## 2. 핵심 설계 — 두 축의 분리와 결합

세션의 다양성은 서로 독립인 두 축으로 구성된다. 이 둘을 분리해 만든 뒤 곱한다.

### 사람 축 (모듈 A) — "누구인가"
- 나이·안정심박·체력 수준이 다른 개인차.
- 효준님(Role B)의 `synth_session`을 재사용(import). 실측 116분 세션의 운동 강도 곡선을 유지하되, 페르소나별 몸(안정심박·체력)에 맞게 심박·SpO₂·이동량을 변형.
- 같은 운동 강도라도 체력이 낮으면 심박이 더 크게 반응하는 생리학적 관계를 반영.

### 상황 축 (모듈 B) — "어떤 상황인가"
- 정상 / 주의 / 경고 / 위험의 위험 수준.
- 본 과제(Role A)에서 신규 작성(`risk_synth`). 세션 후반부에 상황을 주입해 산행이 진행될수록 악화하는 서사를 만든다.
- 규격 위험조건에 정박: 경고=SpO₂ 경고구간(91~94)+과부하, 위험=SpO₂ 위험구간(<90)+과부하+개인편차+이동저하. 근거 없이 위험을 찍지 않으며, 전부 '가상' 계층으로 표기.

### 결합기 — A × B
- 페르소나 10명 × 상황 4종 = 40세션.
- `heat_index`(30.1)·`accident_prior`(0.31) 등 환경·맥락 상수는 실측값을 유지(왜곡 금지). 위험 등급은 생체·이동 신호로만 조절.

### 축을 분리한 이유 (설계 의도)
1. **독립 검증**: 사람 축과 상황 축을 따로 테스트할 수 있다.
2. **실데이터 교체 용이**: iNavi 실데이터 도착 시 사람 축(모듈 A)만 실데이터 로더로 교체하면 되고, 상황 축·결합기는 그대로 재사용된다.
3. **역할 경계 정합**: 효준님과 합의한 "합성 로직 Role A 이관" 방향과 일치. 모듈 A가 이관 대상, 모듈 B가 신규 작성분으로 경계가 명확.

## 3. 파일 구성

| 파일 | 역할 | 비고 |
|---|---|---|
| `synth_sessions.csv` | **최종 산출물**. 40세션 통합 데이터 (4,640행) | window 단계 입력 |
| `sessions_summary.json` | 산출물 요약 리포트 (세션 수·분포·길이 근거 등) | 검증용 |
| `risk_synth.py` | **모듈 B (상황 축)**. 정상/주의/경고/위험 상황 프로파일 및 주입 로직 | Role A 신규 작성 |
| `session_builder.py` | **결합기**. 모듈 A(import) × 모듈 B → 세션 집합 생성 | Role A 신규 작성 |
| `save_sessions.py` | 산출물 저장 스크립트 (통합 CSV + 요약 리포트) | Role A 신규 작성 |
| `../rule_ref.py` | 규칙 공식 **검산용 사본** (상위 폴더 공용). 합성 신호가 목표 등급을 내는지 확인 | 아래 주의 참조 |

### `rule_ref.py`에 대한 주의
규칙 공식 `compute_e1_e2`의 원본(단일 출처)은 효준님 `Model/f1_model.py`다. `rule_ref.py`는 격리 검증 환경에서 상대 import 없이 공식을 참조하기 위한 사본이며, 상수·공식은 원본과 1:1 동일하다. 실제 레포에 통합할 때는 `f1_model`을 직접 import하도록 교체해 이중 관리를 방지한다.

### 모듈 A(효준님 `synth_session`)가 파일에 없는 이유
사람 축은 효준님 `Model/maml/tasks.py`의 `synth_session`을 import해 재사용한다. 코드를 복사하지 않고 참조만 하며, 실데이터 도착 시 이 지점을 실데이터 로더로 교체할 예정이다(이관 트리거 = 실데이터 도착). 따라서 `session_builder.py`·`save_sessions.py` 실행은 `Model.maml` 패키지가 병합된 브랜치·환경을 전제로 한다.

## 4. 산출물 데이터 명세

### `synth_sessions.csv` 컬럼
- **메타**: `session_id`, `persona_name`, `age`, `situation`, `minute_idx`
- **신호 9종**: `hr_mean_bpm`, `hr_ratio_maxhr`, `hr_z_personal`, `spo2_min_pct`, `steps_1min`, `speed_mean_mpm`, `cumulative_min`, `heat_index`, `accident_prior`
- **정답(y) 미포함** — window 단계에서 규칙-오라클(teacher)이 부착

### 규모
- 40세션 × 116행 = **4,640행**
- 상황 분포: 정상 10 / 주의 10 / 경고 10 / 위험 10
- 페르소나: 10명 (나이 30~75, 체력·안정심박 교차)

## 5. 확정 사항 — 세션 길이 116행

전 세션 길이는 **116행(116분)**으로 고정한다.

- **근거**: 실측 뼈대 세션(2023-08-04 05:00~06:55 UTC, 1분 간격 116개 시점)의 길이. `cumulative_min` 0~115로 확인.
- 효준님 `synth_session`이 원본 길이(`n = len(base)`)를 그대로 승계하므로, 결합기가 만든 전 세션이 자동으로 116행.
- 이전에 혼선이 있던 125행은 근거 없는 오류값으로 폐기.

## 6. 검증 결과

### 세션 길이 일관성
- 총 4,640행 = 40 × 116 (정확히 일치)
- 전 세션 116행 확인. 행수 종류 = {116} 단일값. 이상값 없음.

### 상황별 위험 도달 구분
5분 슬라이딩 window(max 대표)로 상황별 등급 분포를 확인한 결과:
- **정상 세션**: 위험 등급 window 0개 (위험 미도달)
- **위험 세션**: 위험 등급 window 다수 존재 (위험 도달)
- 세션의 상황 등급 = 그 세션의 **최고 도달 등급**으로 해석. 세션 안에서 정상→위험으로 자연 악화하며, 이는 실제 산행의 흐름과 부합.
- window 단위 정밀 등급 분포는 window 단계에서 각 window에 정답을 부착할 때 관리.

### 신호 서사 예시 (동일 페르소나 user01, 30세)
- **정상 세션**: 심박 ~71bpm, SpO₂ ~98%, 걸음 ~115 (평온한 산행 유지)
- **위험 세션 후반**: 심박 152→183bpm(비율 0.80→0.97), SpO₂ 90→87%(위험구간 진입), 걸음 7·속도 1(탈진) — 규격 위험조건에 정박한 자연스러운 악화

같은 사람인데 상황만 갈린 것이 "사람 축 × 상황 축" 결합이 실제로 구현되었음을 보여준다.

## 7. 다음 단계 (window 학습셋 구성)

이 세션들을 입력으로 (`../windows/` 참조):
1. 슬라이딩 window(1·3·5·10·13분, 1칸씩 겹침) 적용
2. window 통계 피처(X) 생성 — 구간 평균·최대·최소·표준편차·기울기
3. 규칙-오라클로 정답(y) 부착 — window 대표는 max
4. X/y 별도 컬럼 분리 → 효준님이 바로 학습에 쓸 수 있는 정답 포함 데이터셋

## 8. 미해결·주의 사항

- **가상 데이터임**: 본 산출물은 전부 가상 계층이다. 실데이터(iNavi) 도착 시 사람 축을 교체해 재생성한다.
- **위험군 비율 잠정**: 정상50/주의20/경고20/위험10은 잠정값. 세션 생성 후 위험 감지 성능을 보고 조정 가능.
- **rule_ref 이중 관리**: 레포 통합 시 `f1_model` 직접 import로 교체 필요.
- **정답 미부착 상태**: 본 산출물은 신호(X)만 담긴 세션이다. 정답은 window 단계에서 부착된다.
